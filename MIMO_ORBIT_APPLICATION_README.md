### 二、AI 驱动构建的具体成果（100 词以上）

我正在开发一个“健身陪伴型长效记忆智能体”，核心痛点是很多用户无法持续训练：训练计划难以持续调整、断续锻炼后无法无缝衔接、长期偏好难以被系统记住。目标用户是希望“随时开始训练”的普通用户，以及训练间断的用户（需要阶段化的计划支持）。

项目核心逻辑包含：短期记忆（当前对话上下文）、长期记忆（用户画像与偏好）、检索与融合（从历史对话中检索相关信息并与当前意图融合），形成可持续优化的训练建议；同时提供对比评测脚本用于展示“有无记忆机制”的输出差异。当前已完成的数据与工程成果包括：`data/datal.jsonl` 长时序对话数据集（包含偏好变化与关键事件）、短期/长期记忆与 SQLite 画像模块、轻量向量检索、模板生成与本地模型接入开关、评测脚本与报告生成路径。项目已具备完整 Demo 运行流程与可扩展数据管线。

+ 自检模块（数据完整性、记忆一致性与 API 健康检查）目前处于规划阶段，后续会作为质量保障纳入自动化流程。

我计划使用 **MiMo V2.5 或 MiMo API Token 权益** 在三个方向推进：批量生成更大规模评测数据、提升多轮对话一致性与记忆融合质量、作为 Agent 的核心生成器接入与替换，从而提升输出稳定性与可控性，并完善长时程评测基准。
